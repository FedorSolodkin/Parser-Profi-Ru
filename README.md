Profi.ru Parser - Парсер заказов с биржи услуг

ОПИСАНИЕ:
Автономный парсер для мониторинга новых заказов на Profi.ru.
Работает через Selenium, авторизуется по логину/паролю,
отправляет уведомления в Telegram при появлении новых заказов.

ОСОБЕННОСТИ:
- Автоматический вход по логину и паролю
- Фильтрация заказов: обычные и карусельные (доп.)
- Отправка в Telegram: обычные со звуком, доп. без звука
- Защита от дублей: сохранение ID обработанных заказов
- Автообновление страницы каждые 30 секунд
- Работа в headless-режиме на сервере (VPS)

ТРЕБОВАНИЯ:
- Python 3.8+
- Google Chrome / Chromium
- pip install selenium webdriver-manager aiogram python-dotenv
- VPS с Ubuntu (для работы 24/7)

УСТАНОВКА:

1. Создайте папку проекта:
   git clone https://github.com/FedorSolodkin/Parser-Profi-Ru

2. Создайте виртуальное окружение:
   python3 -m venv venv
   source venv/bin/activate  # Linux/Mac
   # или venv\Scripts\activate  # Windows

3. Установите зависимости:
   pip install selenium webdriver-manager aiogram python-dotenv

4. Создайте файл .env с данными для входа:
   PROFI_LOGIN=ваш_логин_или_телефон
   PROFI_PASSWORD=ваш_пароль

5. Скопируйте main.py в папку проекта.

НАСТРОЙКИ (в main.py):

TELEGRAM_TOKEN = "ваш_токен_бота"     # От @BotFather
CHAT_ID = "123123123"            # ID чата или группы
TARGET_URL = "https://profi.ru/backoffice/n.php"
CHECK_INTERVAL = 30                   # Интервал проверки в секундах
DB_FILE = "seen_orders.json"          # Файл с историей заказов

КАК ПОЛУЧИТЬ TELEGRAM TOKEN И CHAT_ID:

1. Токен бота:
   - Напишите @BotFather в Telegram
   - Отправьте /newbot
   - Следуйте инструкциям, скопируйте токен

2. Chat ID:
   - Добавьте бота в чат/группу
   - Напишите любое сообщение
   - Узнайте ID через @userinfobot или @getidsbot
   - Для группы ID обычно начинается с минуса: -100...

КАК РАБОТАЕТ ПАРСЕР:

1. Запускает браузер (Chrome/Chromium)
2. Открывает TARGET_URL
3. Вводит логин → Tab → пароль → Enter
4. Проверяет, что вход успешен (по URL)
5. Ищет карточки заказов по селектору:
   a[data-testid*='order-snippet']
6. Для каждой карточки:
   - Извлекает ID, заголовок, описание, ссылку
   - Проверяет, не обрабатывался ли заказ ранее
   - Определяет тип: обычный или карусельный
   - Отправляет в Telegram с нужными настройками
7. Ждёт CHECK_INTERVAL секунд и обновляет страницу

ТИПЫ ЗАКАЗОВ:

Обычный заказ:
- Пометка: 🔥 Новый заказ!
- Отправка: со звуком
- Условие: есть описание (p[class*='sc-gaeugI']) и не в карусели

Доп. заказ (карусель):
- Пометка: 🔔 ДОП. ЗАКАЗ
- Отправка: без звука (disable_notification=True)
- Условие: внутри #flatlist-container-CAROUSEL-content

ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ:

1. Ошибка ChromeDriver version mismatch:
   - Обновите Chrome: sudo apt upgrade chromium
   - Скачайте подходящий драйвер вручную
   - Или используйте системный: /usr/bin/chromedriver

2. Не входит в аккаунт:
   - Проверьте логин/пароль в .env
   - Посмотрите скриншот 04_after_enter.png
   - Возможно, нужна капча или СМС — пройдите вручную один раз

3. Не находит карточки:
   - Сайт мог обновиться — проверьте селекторы через F12
   - Убедитесь, что вы в нужном разделе (фильтры применены)

4. Блокировка по IP:
   - Добавьте прокси в chrome_options
   - Увеличьте CHECK_INTERVAL до 60-120 секунд

5. Память заканчивается на сервере:
   - Добавьте swap-файл
   - Или увеличьте RAM на тарифе VPS

6. Бот падает с ошибкой:
   - Проверьте логи: tail -f bot.log
   - Убедитесь, что все зависимости установлены
